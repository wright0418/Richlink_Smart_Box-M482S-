# TYPE 值修正摘要

## 修正內容

已將所有 TYPE 定義值修正為正確的數值：

| TYPE | 修正前 | 修正後 | 說明 |
|------|-------|-------|------|
| RL_TYPE_GET | 0x01 | **0x00** | 讀取 DI（本地處理） |
| RL_TYPE_SET | 0x02 | **0x01** | 控制 DO（本地處理） |
| RL_TYPE_RTU | 0x03 | **0x02** | Modbus RTU 透傳 |

## 修改的檔案

1. ✅ `mesh_modbus_agent.c` - 程式碼中的 #define 定義
2. ✅ `MESH_AGENT_TYPE_TEST_GUIDE.md` - 測試文件中所有範例與說明

## 封包格式（修正後）

### GET - 讀取 DI
```
請求：82 76 00 00
回應：82 76 00 80 00 [DI]
```

### SET - 控制 DO
```
請求：82 76 01 00 [VALUE]
回應：82 76 01 80 00 [DO]
```

### RTU - Modbus 透傳
```
請求：82 76 02 [Modbus_ADU]
回應：82 76 02 [Modbus_Response無CRC]
```

## 現在可以正常使用

**RTU 現在會正確回傳 sensor 資料**，因為 TYPE 值已修正為 0x02，不會再與 GET(0x00) 或 SET(0x01) 的 STATUS_OK(0x80) 格式混淆。

---

**建議測試步驟**：

1. 測試 GET：發送 `82 76 00 00`，確認收到 DI 狀態
2. 測試 SET：發送 `82 76 01 00 01`，確認 DO 被控制並回應狀態
3. 測試 RTU：發送原本的 RTU 封包（Type 改為 0x02），確認收到 sensor 資料

例如 RTU 讀取：
```
原本（錯誤）：82 76 03 01 03 00 00 00 01 [CRC]
修正後：    82 76 02 01 03 00 00 00 01 [CRC]
                   ^^
                   改為 02
```
